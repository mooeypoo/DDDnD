# Strategy

## Classes and Archetypes

Each player character represents a software architecture archetype. Your choice affects which cards are available and how stakeholders react.

## Card Families

Cards fall into several categories based on the architecture decision they represent:

- **Refactoring** - Improve code clarity and structure
- **Infrastructure** - Build out supporting systems
- **Team Dynamics** - Manage developer morale and communication
- **Process** - Establish workflows and governance
- **Fixes** - Address immediate bugs and issues
